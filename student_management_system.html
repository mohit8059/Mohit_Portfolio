Student Management System

A complete desktop-based management system built using Python, Tkinter, MySQL and OOP concepts to manage student records, attendance, fees, courses, reports and role-based access.

View GitHub Back to Portfolio

Project Overview

The Student Management System is designed to help institutes, schools and coaching centers manage their daily academic and administrative tasks from one application.

This project includes student registration, attendance management, fee tracking, course management, user login system, role-based access and report generation. The main goal of this project is to reduce manual paperwork and make student data easy to manage, search and update.

This project helped me understand real-world Python application development, database connectivity, GUI design, CRUD operations, validation and modular project structure.

Key Features

Student Management

Add, update, delete and search student records with roll number, name, course, mobile number and address details.

Attendance System

Mark student attendance as Present, Absent or Leave and view attendance records using date and student filters.

Fees Management

Track total fees, paid fees, pending amount and generate simple fee records for students.

Course Management

Manage courses, course duration, fees and connect students with their selected course.

Role-Based Login

Admin and Teacher roles with different access permissions for better security and workflow control.

Reports

Generate and view reports related to students, attendance, fees and course records.

Main Modules

1. Login Module

Secure login system for Admin and Teacher users. Different roles can access different parts of the application.

2. Dashboard Module

Shows important summary cards like total students, total courses, today attendance and pending fees.

3. Student Module

Handles student registration, update, delete, search and student profile management.

4. Attendance Module

Used to mark and view daily attendance records using student and date filters.

5. Fees Module

Tracks total fees, paid fees, pending fees and payment records of students.

6. Reports Module

Provides structured reports for student details, attendance and fee status.
